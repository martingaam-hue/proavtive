2. **Add `prepare` script to `package.json`:**
   ```json
   "scripts": {
     ...existing...,
     "prepare": "lefthook install"
   }
   ```
   This makes `pnpm install` auto-install git hooks on every clone (the GSD-standard dev-onboarding pattern). If `.git/` is missing (e.g. running from a tarball), `lefthook install` is a no-op and will not fail `pnpm install`.

3. **Create `.gitleaks.toml`** — minimal config that extends the default ruleset (per "Claude's Discretion" in CONTEXT: "use sensible default ruleset, extend only if Phase 0 turns up real false-positives"):
   ```toml
   # gitleaks config — extends default rules only.
   # Extend the `[allowlist]` block when a Phase 0 scan turns up a confirmed false-positive.
   title = "proactive-gitleaks"

   [extend]
   useDefault = true

   [allowlist]
   description = "Files where high-entropy strings are expected (lockfiles, planning docs)"
   paths = [
     '''pnpm-lock\.yaml''',
     '''package-lock\.json''',
     '''yarn\.lock''',
     '''\.planning/.*\.md''',
   ]
   ```
   NOTE: the `.env.example` file is NOT allowlisted — it has empty values, so gitleaks will pass it clean. Allowlisting it would be a mistake (we WANT gitleaks to scan it in case someone accidentally pastes a real value).

4. **Create `lefthook.yml`** with three hooks: `pre-commit`, `commit-msg`, and `pre-push` (pre-push as a defensive duplicate of gitleaks against the full diff being pushed):
   ```yaml
   # Defense-in-depth layer 1 (D-05) and commit discipline (D-11).
   # Hooks can be bypassed with `git commit --no-verify` — CI gitleaks (Plan 04) is the ultimate catch (D-07).

   # Preflight: warn if gitleaks binary is missing.
   pre-commit:
     parallel: true
     commands:
       gitleaks-check:
         # Scans staged content only. --redact hides secret values in log output.
         run: |
           if ! command -v gitleaks >/dev/null 2>&1; then
             echo "ERROR: gitleaks not installed. Install via: brew install gitleaks (macOS) or https://github.com/gitleaks/gitleaks"
             exit 1
           fi
           gitleaks protect --staged --redact --verbose --config .gitleaks.toml
       lint-staged:
         run: pnpm exec lint-staged

   commit-msg:
     commands:
       commitlint:
         run: pnpm exec commitlint --edit {1}

   pre-push:
     commands:
       gitleaks-full:
         # Defensive — scans the full diff being pushed, catches anything committed with --no-verify.
         run: |
           if ! command -v gitleaks >/dev/null 2>&1; then
             echo "WARN: gitleaks not installed — skipping pre-push scan. CI will catch anything that slips through."
             exit 0
           fi
           gitleaks detect --redact --verbose --config .gitleaks.toml
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer editor → git staging area → `.git/hooks/` | Staged content passes through local hooks; anything rejected here never enters history |
| developer shell → `--no-verify` flag | Documented escape hatch that bypasses local hooks — must be caught downstream (Plan 04 CI gitleaks) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-06 | Information disclosure | committed secrets | mitigate | Layer 1: lefthook pre-commit + pre-push runs gitleaks against staged / pushed diff with default ruleset. Redacted output prevents the hook's own logs from leaking the secret. |
| T-00-07 | Tampering | `--no-verify` bypass (D-05/D-07) | accept + mitigate downstream | Bypass is documented behaviour (engineer needs the escape hatch occasionally). Plan 04 CI gitleaks is mandatory and cannot be bypassed — that is the ultimate catch. |
| T-00-08 | Tampering | gitleaks binary not installed on developer machine | mitigate | `lefthook.yml` pre-commit fails loudly with install instructions; pre-push degrades to a warning (doesn't block pushes) because CI will still catch it. |
| T-00-09 | Repudiation | non-conventional commit messages make changelogs / blame hard to read | mitigate | commit-msg hook runs commitlint with `@commitlint/config-conventional`. Blocks all non-conventional subjects locally. |
</threat_model>
