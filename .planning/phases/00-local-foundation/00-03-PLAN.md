3. **Rewrite `next.config.js`** to add the D-15 noindex header on non-production deployments. Use this exact shape so Plan 05 can wrap the default export with `withSentryConfig` cleanly:

   ```js
   // next.config.js
   // Phase 0 / Plan 03 — adds X-Robots-Tag: noindex, nofollow on all non-production deploys (D-15).
   // Plan 05 will wrap the default export with withSentryConfig(...) — keep shape wrap-friendly.

   /** @type {import('next').NextConfig} */
   const nextConfig = {
     reactStrictMode: true,

     async headers() {
       // VERCEL_ENV is auto-injected by Vercel at runtime:
       //   'production'  — the live custom domain (Phase 10)
       //   'preview'     — every PR / branch preview
       //   'development' — `vercel dev` locally (rare; `pnpm dev` uses plain Next which sets nothing)
       // We block indexing for anything that is NOT production — belt-and-braces behind Deployment Protection (D-14).
       const isProd = process.env.VERCEL_ENV === 'production';

       if (isProd) {
         // Production: no noindex header. Real robots.txt ships in Phase 7 (SEO-03).
         return [];
       }

       return [
         {
           source: '/:path*',
           headers: [
             { key: 'X-Robots-Tag', value: 'noindex, nofollow' },
           ],
         },
       ];
     },
   };

   module.exports = nextConfig;
   ```

   Rationale (per D-15): Deployment Protection gates humans, but if an editor ever disables Deployment Protection accidentally (e.g. to share a preview with the client) the `X-Robots-Tag` still tells crawlers to stay out. Belt AND braces.

   ```json
     "installCommand": "pnpm install --frozen-lockfile"
   }
   ```
   This file is COMMITTED. It makes Vercel's behaviour explicit rather than inferred — helpful when debugging future build diffs.
