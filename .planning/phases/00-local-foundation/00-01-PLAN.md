<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer machine → git → GitHub | Local commits cross into a shared remote; leaked secrets here propagate to GitHub history and any cloned fork |
| developer machine → `.env.local` → Next.js runtime | Real secret values live only on the developer machine; any mis-gitignore pattern risks accidentally committing them |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-01 | Information disclosure | `.env` files | mitigate | `.gitignore` includes `.env`, `.env*.local`, `.env.development`, `.env.production` with explicit `!.env.example` negation to prevent the example file being accidentally ignored; `.env.example` values are all empty (verified by acceptance criterion) |
| T-00-02 | Information disclosure | `.env.example` itself | accept | `.env.example` lists variable NAMES only (empty values) — names alone are not secrets, and exposing them is the point (onboarding aid). Risk accepted. |
| T-00-03 | Tampering | `package.json` pinning drift | mitigate | `packageManager` field activates Corepack at exact pinned pnpm version on local / CI / Vercel; `pnpm-lock.yaml` committed; `engines` enforced via `pnpm install` strict-engines config (default in pnpm 9+) |
| T-00-04 | Spoofing | unauthorised commits to main | mitigate (deferred) | Branch protection is Plan 04's job; Plan 01 only creates the repo. Plan 06 verifies protection end-to-end. |
| T-00-05 | Information disclosure | dependency drift (future CVEs in scaffold deps) | mitigate | Dependabot configured with weekly npm + GitHub Actions updates; CVE gate enforced in Phase 9 MIG-03 |
</threat_model>
