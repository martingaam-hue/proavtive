# Plan 09-02: Security Headers in next.config.ts

## Context

Phase 0 added a conditional `headers()` function in `next.config.ts` that sets `X-Robots-Tag: noindex, nofollow` on non-production deployments. Phase 9 extends this function to add six security headers on ALL routes: HSTS, X-Content-Type-Options, X-Frame-Options, Referrer-Policy, Permissions-Policy, and a Content-Security-Policy covering all Phase 0–8 resource origins.

The existing logic must be preserved: `X-Robots-Tag` still only applies to non-production. The new security headers apply on ALL environments (production and preview) because security posture should not be weaker in preview.

The CSP is defined as a TypeScript constant `CSP_HEADER` above the config object for readability. `'unsafe-inline'` is required for `script-src` (GTM) and `style-src` (Tailwind). A future hardening pass can introduce nonces post-launch.

**No new npm dependencies are required.** All changes are in `next.config.ts` only.

## T02 — Replace the headers() function body in next.config.ts

**File:** `next.config.ts`

**Current `headers()` function** (inside `nextConfig`):
```typescript
  async headers() {
    const isProd = process.env.VERCEL_ENV === "production";

    if (isProd) {
      return [];
    }

    return [
      {
        source: "/:path*",
        headers: [{ key: "X-Robots-Tag", value: "noindex, nofollow" }],
      },
    ];
  },
```

**Replace with:**
```typescript
  async headers() {
    const isProd = process.env.VERCEL_ENV === "production";

    // Security headers applied to ALL routes on ALL environments.
    // HSTS includeSubDomains covers hk.* and sg.* once domain is live (Phase 10).
    // X-Frame-Options SAMEORIGIN allows Sanity Studio iframe (same-origin embed).
    const securityHeaders = [
      {
        key: "Strict-Transport-Security",
        value: "max-age=31536000; includeSubDomains; preload",
      },
      {
        key: "X-Content-Type-Options",
        value: "nosniff",
      },
      {
        key: "X-Frame-Options",
        value: "SAMEORIGIN",
      },
      {
        key: "Referrer-Policy",
        value: "strict-origin-when-cross-origin",
      },
      {
        key: "Permissions-Policy",
        value: "camera=(), microphone=(), geolocation=(), payment=()",
      },
      {
        key: "Content-Security-Policy",
        value: CSP_HEADER,
      },
    ];

    if (isProd) {
      // Production: security headers only — no noindex.
      return [
        {
          source: "/(.*)",
          headers: securityHeaders,
        },
      ];
    }

    // Non-production (preview / dev): security headers + noindex.
    return [
      {
        source: "/(.*)",
        headers: [
          ...securityHeaders,
          { key: "X-Robots-Tag", value: "noindex, nofollow" },
        ],
      },
    ];
  },
```

**Key changes from the original:**
- `source` pattern changed from `"/:path*"` to `"/(.*)"` — the `(.*)` form is the Vercel-recommended catch-all for headers; `/:path*` works but `(.*)` is more explicit and consistent with the UI-SPEC §5.1 contract.
- Security headers always applied; `X-Robots-Tag` still only on non-production.
- The `if (isProd) { return []; }` early-exit is replaced by a conditional that still produces the correct output.
</task>
