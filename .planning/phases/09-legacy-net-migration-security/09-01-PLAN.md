# Plan 09-01: CVE Remediation and CI Audit Gate

## Context

`pnpm audit --prod` shows 1 HIGH CVE: `path-to-regexp` reachable via `shadcn > @mcp/sdk > express > router`. The `shadcn` package is listed in `dependencies` (not `devDependencies`) so `--prod` audit picks it up. Adding a `pnpm.overrides` block forces `path-to-regexp >= 8.4.0` across the tree. The 9 remaining HIGH CVEs in the full audit all trace to the `vercel` devDependency — excluded by `--prod` and documented in code. After the fix, `pnpm audit --prod --audit-level high` must exit 0. That command is then added as required check 6 in `.github/workflows/ci.yml` (after Unit tests, before Gitleaks) to prevent future regressions.

**Pitfall 6 avoidance:** After adding overrides, `pnpm install` must be run locally to regenerate `pnpm-lock.yaml`. Both files are committed together so CI's `--frozen-lockfile` continues to pass.
