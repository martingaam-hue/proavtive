# Plan 08-03 — NAP Canonical Venue Source of Truth

## Context

Phase 8 requires a single source of truth for all three venue NAP (Name, Address,
Phone) records. Currently, address and phone strings are hardcoded in individual
location page files and potentially in layout footer sections. Any divergence
between the site's footer, location page, JSON-LD schema, and the GBP listing
reduces NAP consistency signals.

This plan creates `lib/venues.ts` and migrates all hardcoded NAP strings to import
from it. The `lib/venues.test.ts` scaffold (created in plan 08-01 in RED state)
turns GREEN when this plan completes.
