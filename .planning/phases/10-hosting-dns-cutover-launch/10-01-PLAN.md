HUMAN-ACTION — cannot be automated.

1. Check current DNSSEC status:
   ```bash
   dig +dnssec +short DS proactivsports.com @8.8.8.8
   ```
   - If the command returns nothing → DNSSEC is NOT active. Record "DNSSEC: INACTIVE" in DNS-RECORDS.md. No action needed.
   - If the command returns DS records → DNSSEC IS active. Must be disabled before NS change.

2. If DNSSEC is active:
   a. Log into current registrar → Domain settings → DNSSEC → Disable
   b. Note: DS record TTL can be up to 24h — wait before proceeding with NS change
   c. Verify disabled:
      ```bash
      # Re-run after registrar confirms disabled
      dig +dnssec +short DS proactivsports.com @8.8.8.8
      # Should return empty
      ```

3. Record DNSSEC status in DNS-RECORDS.md under "Open Audit Items":
   - [ ] DNSSEC status confirmed at current registrar: [ACTIVE | INACTIVE]
   - [ ] DNSSEC disabled (if was active)

BLOCKING: NS change cannot proceed until DNSSEC is confirmed inactive.

HUMAN-ACTION — cannot be automated (requires Cloudflare Dashboard access).

1. Log into Cloudflare Dashboard → Add a domain → Enter `proactivsports.com` → Continue.

2. Select Free plan (DNS management only — WAF is on Vercel side).

3. Cloudflare will auto-scan existing DNS records. IMPORTANT: This scan is incomplete by design. Do NOT rely on it alone.
   - Review the auto-detected records
   - Manually add any missing records from the zone export completed in Task 3
   - Specifically verify: MX records, SPF TXT, DKIM CNAME/TXT, DMARC TXT, any third-party TXT records

4. Set ALL records pointing to Vercel as DNS-only (gray cloud — NOT orange cloud):
   - The proxy icon must show gray/cloud with a line through it, NOT the orange cloud
   - This is critical: orange cloud breaks Vercel ACME cert provisioning

5. Configure Cloudflare SSL/TLS → Overview → Set to "Full (strict)":
   - Even though gray-cloud mode means Cloudflare SSL is not in the traffic path
   - Defensive config: prevents redirect loops if orange-cloud is ever accidentally enabled

6. Note the Cloudflare nameservers assigned to this zone:
   - They appear on the "Review DNS records" screen
   - Format: `xxxx.ns.cloudflare.com` and `yyyy.ns.cloudflare.com`
   - Record these in DNS-RECORDS.md (needed for Task 4 in Plan 10-04)

7. DO NOT change nameservers at the registrar yet. The zone is created but NS change happens in Plan 10-04.

Record in DNS-RECORDS.md:
- Cloudflare zone created: [date]
- Assigned NS pair: [ns1] and [ns2]
- SSL/TLS mode: Full (strict) confirmed
