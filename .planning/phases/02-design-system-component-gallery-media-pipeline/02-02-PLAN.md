`next/font/google` signature (verified against Next.js 15.5 docs):

```typescript
import { Unbounded, Manrope, Baloo_2 } from 'next/font/google';

export const unbounded = Unbounded({
  subsets: ['latin'],
  weight: ['400', '700', '800'],
  variable: '--font-unbounded',
  display: 'swap',
});
```

Notes:
- Google Font function names are the family name with spaces replaced by underscores (`Baloo 2` → `Baloo_2`).
- `subsets` is REQUIRED by `next/font/google`. `subsets: ['latin']` covers the entire site (no `zh-HK` content in v1.0 per PROJECT.md Out of Scope).
- `weight` must match available weights on Google Fonts. Verified available: Unbounded 200–900, Manrope 200–800, Baloo 2 400–800.
- `display: 'swap'` is non-negotiable per UI-SPEC §1.7.
- `adjustFontFallback` defaults to `true` on `next/font/google` — Google's metric database handles it automatically, unlike `next/font/local` which needs explicit `adjustFontFallback: 'Arial'`.
- No `fallback` chain needed — Next.js generates a metric-matched fallback font from the real font's metrics.

Font weights per UI-SPEC §1.7 (amended):
- Unbounded: 400 Regular + 700 Bold + 800 ExtraBold
- Manrope: 400 Regular + 500 Medium + 700 Bold
- Baloo 2: 400 Regular + 500 Medium + 700 Bold
