## T2: HUMAN-ACTION — Manual Lighthouse measurements on primary page set

<read_first>
- `docs/phase-7-lighthouse-checklist.md` (created in Plan 07-05 T4) — the measurement procedure
</read_first>

<action>
**This task requires human action in a browser on the latest Vercel preview deploy.**

Wait for the latest preview deployment to complete on Vercel, then measure each page in the primary page set.

**Measurement procedure** (from `docs/phase-7-lighthouse-checklist.md`):
1. Open Chrome DevTools → Lighthouse tab
2. Set: Mobile, all 4 categories (Performance, Accessibility, Best Practices, SEO)
3. Set throttling: Slow 4G
4. Disable browser extensions
5. Clear cache before each measurement (Ctrl+Shift+R)
6. Run audit on cold cache

**Pages to measure:**

Root:
- `https://{preview-url}/` (root homepage with appropriate Host header or preview-alias URL)

HK (use `hk.{preview-url}` or configured HK preview alias):
- `/` (HK homepage)
- `/wan-chai/`
- `/cyberport/`
- `/gymnastics/`
- One gymnastics sub-page (e.g., `/gymnastics/toddlers/`)

SG (use `sg.{preview-url}` or configured SG preview alias — only if Phase 5 is complete):
- `/` (SG homepage)
- `/katong-point/`
- `/weekly-classes/`

**Thresholds (all must be ≥ 95):**
- Performance: ≥ 95
- Accessibility: ≥ 95
- Best Practices: ≥ 95
- SEO: ≥ 95

**CWV thresholds:**
- LCP: < 2.5s
- INP: < 200ms
- CLS: < 0.1

**If a page scores below 95:**

1. Note the specific failing metric (Performance, Accessibility, Best Practices, or SEO)
2. Note the specific Lighthouse opportunities/diagnostics listed
3. Common fixes:
   - Performance < 95 with high LCP: Check `sizes` prop on hero image (07-05-T1 should have fixed this)
   - Accessibility < 95: Check axe-core findings in DevTools → run axe in console
   - SEO < 95: Check that JSON-LD is present and robots/canonical are correct
4. Fix and re-deploy before proceeding

**Record all scores** in `docs/phase-7-verification.md` (created in T4).

**Acceptable exclusions:**
- `/_design/` gallery page: expected 55–81 due to Mux bootup. Not a primary page.
- Any page not yet built (Phase 5/6 stubs): document as "Page not yet built — Phase X prerequisite"
</action>

<acceptance_criteria>
- All primary pages in the test set have been measured
- Scores are documented in `docs/phase-7-verification.md` (T4 creates the file)
- Pages meeting ≥ 95 on all 4 Lighthouse categories are marked as PASS
- Any pages below 95 have a specific remediation plan or are documented as acceptable exclusions
</acceptance_criteria>
</task>
